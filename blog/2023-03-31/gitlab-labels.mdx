# Gitlab中常见的标签

`Gitlab`中的标签是一个非常有用的功能, 该功能可以帮助我们更好的管理`ISSUE`.<br/>
下面介绍一下`Gitlab`中常见的标签的定义和其使用希望能够对你有所帮助.<br/>

<!--truncate-->

## 标签的定义

下面介绍常见的一些标签的定义, 以便我们更好的使用.

|      名称       |                   颜色                   | 16进制代码  |    描述    |             使用时机              |
|:-------------:|:--------------------------------------:|:-------:|:--------:|:-----------------------------:|
|      Bug      | <span style="color:#FF0000">红色</span>  | #FF0000 |  错误或需修复  |     当发现或报告一个问题时，可以使用这个标签      |
|   Duplicate   | <span style="color:#808080">灰色</span>  | #808080 | 重复的issue |   当发现或报告一个已经存在的问题时，可以使用这个标签   |
|    Feature    | <span style="color:#0000FF">蓝色</span>  | #0000FF |  新功能或需求  |   当提出或实现一个新的功能或需求时，可以使用这个标签   |
|     Fixed     | <span style="color:#00FF00">绿色</span>  | #00FF00 |  已修复或完成  |    当解决或完成一个问题或任务时，可以使用这个标签    |
|    Invalid    | <span style="color:#000000">黑色</span>  | #000000 |  无效或不合理  | 当发现或报告一个不符合规范或逻辑的问题时，可以使用这个标签 |
|   Proposal    | <span style="color:#800080">紫色</span>  | #800080 |  建议或想法   |    当提出或讨论一个建议或想法时，可以使用这个标签    |
|     Task      | <span style="color:#FFA500">橙色</span>  | #FFA500 |  任务或工作   |    当分配或执行一个任务或工作时，可以使用这个标签    |
|    Wontfix    | <span style="color:#A52A2A">棕色</span>  | #A52A2A | 不会或不需修复  |    当决定不处理或忽略一个问题时，可以使用这个标签    |
|  Improvement  | <span style="color:#FFC0CB">粉色</span>  | #FFC0CB |  改进或优化   |  当对一个功能或需求进行改进或优化时，可以使用这个标签   |
|  Enhancement  | <span style="color:#ADD8E6">淡蓝色</span> | #ADD8E6 |  增强或扩展   |  当对一个功能或需求进行增强或扩展时，可以使用这个标签   |
|   Question    | <span style="color:#FFFF00">黄色</span>  | #FFFF00 |  疑问或询问   |   当对一个问题或任务有疑问或询问时，可以使用这个标签   |
|  Help wanted  | <span style="color:#FF00FF">紫红色</span> | #FF00FF | 需要帮助或协作  |    当需要其他人的帮助或协作时，可以使用这个标签     |
| Documentation | <span style="color:#90EE90">浅绿色</span> | #90EE90 | 需要或涉及文档  |   当需要或涉及文档的编写或更新时，可以使用这个标签    |
